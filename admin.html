    📚 Guide d'Utilisation Complet

    🏗️ Architecture du Systeme

    Le site Little Smart Genius fonctionne avec un pipeline automatise :

    • GitHub Actions — Execute les scripts Python automatiquement (generation, build, maintenance)
    • auto_blog_v6_ultimate.py — Genere les articles avec 7 agents DeepSeek AI + images via Pollinations (klein-large)
    • build_articles.py — Convertit les JSON en pages HTML + genere articles.json, sitemap, search_index
    • rebuild_blog_pages.py — Genere les pages blog.html, blog-2.html, etc.
    • Cloudflare Workers + GitHub Pages — Heberge le site statique (GitHub Pages) et les fonctions API (Cloudflare Workers: admin-api, contact, freebie-email, subscribe)

    📊 Onglet Overview

    Vue d'ensemble rapide du site :

    • Articles — Nombre total d'articles publies
    • Images — Nombre total d'images generees
    • Instagram — Nombre de posts Instagram generes
    • Health Score — Score de sante global (fichiers manquants, erreurs)
    • Keywords restants — Nombre de mots-cles encore disponibles pour la generation
    • Semaine — Semaine actuelle du calendrier editorial
    • Problemes — Nombre total de problemes detectes
    • Categories — Repartition des articles par categorie

    📝 Onglet Articles

    Liste detaillee de tous les articles avec :

    • Titre — Titre complet de l'article
    • Cat — Categorie (Math, Logic, Visual Perception, etc.)
    • Date — Date de publication
    • Cover — Image de couverture (1 = presente, 0 = manquante)
    • Imgs — Images de contenu (X/4 = X images sur 4 attendues)
    • IG — Post Instagram genere (✅/❌)
    • JSON — Fichier post JSON present (✅/❌)
    • Status — Etat global (ok/warning/error)
    • 👁️ Voir — Ouvre l'article dans un nouvel onglet
    • 🔍 Deep Scan — Lance l'analyse SEO approfondie de cet article
    • 🗑️ Supprimer — Supprime l'article + toutes ses ressources (images, JSON, IG)

    📰 Onglet Blog — Deep Scan

    Analyse SEO approfondie de chaque article :

    • Score SEO — Note sur 100% basee sur title, meta description, H1, H2, OG tags, canonical, alt text
    • Images — Nombre total + images manquantes (fichier non trouve sur le serveur)
    • Liens internes — Nombre de liens vers d'autres articles du site
    • Mots — Nombre de mots dans l'article (ideal : 1500-3000)

    Boutons :

    • 🔍 Lancer le Scan — Analyse tous les articles (15-30 secondes)
    • 🖼️ Fix Images — Regenere les images manquantes via Pollinations AI
    • 🏗️ Rebuild Blog — Reconstruit les pages blog avec les derniers articles

    ⚡ Onglet Generator

    Declenchement a distance des workflows GitHub Actions :

    Bouton Action Description
    Keyword Article generate-keyword Genere 1 article SEO base sur le prochain mot-cle non utilise de data/keywords.txt
    Product Article generate-product Genere 1 article review pour un produit TpT aleatoire non encore couvert
    Freebie Article generate-freebie Genere 1 article tutoriel pour un freebie aleatoire
    Batch Complet generate-batch Genere 3 articles d'un coup : 1 keyword + 1 product + 1 freebie

    Chaque article genere inclut : 1 image de couverture + 3 images de contenu + 1 post Instagram + SEO optimise + liens internes

    🏥 Onglet Health Check

    Verification automatique de l'integrite des fichiers :

    • Verifie que chaque article HTML a son JSON, ses images, et son post Instagram
    • Score 100% = aucun fichier manquant
    • Les problemes sont listes avec leur type (erreur, warning, info)
    • Fix : Rebuild corrige la plupart des problemes automatiquement

    🔑 Onglet Topics

    Gestion des sujets pour la generation d'articles :

    • Keywords — Liste des mots-cles SEO. Les mots barres ont deja ete utilises. Ajoutez-en via l'editeur ci-dessus.
    • Produits — Liste des produits TpT ayant deja un article
    • Freebies — Liste des freebies ayant deja un article

    Editeur de Keywords :

    • Ecrivez un keyword par ligne dans la zone de texte
    • Les lignes commencant par # sont des commentaires (categories)
    • Cliquez Sauvegarder pour envoyer au serveur via GitHub API
    • Ou importez un fichier .txt avec le bouton Importer

    📋 Onglet Runs

    Historique des 10 dernieres executions GitHub Actions :

    • 🟢 = succes, 🔴 = echec, 🟡 = en cours
    • Cliquez ↗ pour voir les logs detailles sur GitHub
    • Un run prend generalement 3-5 minutes pour un article et 8-12 minutes pour un batch

    📅 Workflow Quotidien Recommande

    1. Ouvrez le dashboard et verifiez l'Overview
    2. Lancez un Batch (3 articles) depuis l'onglet Generator
    3. Attendez 8-12 min, suivez la progression dans l'onglet Runs
    4. Verifiez les nouveaux articles dans l'onglet Articles
    5. Lancez un Deep Scan pour verifier le SEO
    6. Si des images manquent, cliquez Fix Images
    7. Verifiez le Health Score — objectif : 100%

    🔧 Troubleshooting

    Probleme Solution
    Images manquantes Onglet Build → Fix Images. Si persiste, verifiez les logs dans Runs
    Article sans contenu Le workflow a peut-etre echoue. Verifiez Runs → ↗ pour les logs GitHub
    Score SEO bas (<70%) Deep Scan identifie les problemes. Souvent : meta description manquante ou title trop long.
    Keywords epuises Ajoutez de nouveaux keywords dans Topics → Editeur de Keywords
    Health Score bas Lancez Build Site ou Full Rebuild pour regenerer les fichiers manquants
    Workflow bloque Verifiez les Runs. Si bloque >15min, relancez depuis GitHub Actions directement
